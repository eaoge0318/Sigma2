# 主視窗背景更新修復

## ✅ 問題原因

當您切換到彈出視窗進行操作時，瀏覽器會判斷主視窗變為「背景視窗」。
為了節省電腦電量，現代瀏覽器（Chrome/Edge）會強制降低背景視窗的運作頻率（稱為 Background Throttling），導致圖表更新停止或變得非常慢。

這並不是程式當機，而是瀏覽器的省電保護機制。

## 🛠️ 已執行的修復

我使用了 **Web Worker** 技術來繞過這個限制：

1. **獨立計時器**：創建了一個獨立於主線程的「後臺計時器」。
2. **強制喚醒**：這個計時器有權限在背景運行，它會每 1.5 秒「喚醒」主視窗一次，強制執行 `updateDashboard()`。

## 📝 驗證效果

1. **強制清除快取**：按 `Ctrl + F5` 重新整理頁面。
2. **操作**：
   - 點擊 "Auto Play" 讓圖表開始跑。
   - 打開 AI 彈出視窗。
   - 在彈出視窗中打字。
3. **觀察**：即便您的焦點在彈出視窗，主畫面的圖表現在應該會**持續流暢更新**，不再卡頓或停止。

這是一個高級優化，確保您的監控面板擁有像是「原生應用」一樣的後台運行能力！
