# 如何查看瀏覽器日誌 - 簡易圖文教學

## 為什麼要查看日誌？
日誌會告訴我們程式在執行時發生了什麼，幫助找出問題所在。

## 📋 步驟說明

### 步驟 1：打開開發者工具
在即時看板頁面，按鍵盤上的 **F12** 鍵
（或者按 Ctrl + Shift + I）

會出現一個新的視窗區域（通常在瀏覽器底部或右側）

### 步驟 2：切換到 Console 標籤
在開發者工具視窗中，找到並點擊 **Console** 標籤
（通常在最上方的標籤列，與 Elements、Network 等並列）

### 步驟 3：執行測試
現在在 Console 視窗保持開啟的狀態下：

1. **重新整理頁面**（按 F5）
2. **等待 2-3 秒**
3. **觀察 Console** 應該會看到：
   ```
   Initializing Dashboard Controls...
   📂 File select element: 找到
   🤖 Model select element: 找到
   ✅ 檔案選單事件監聽器已綁定
   ✅ 模型選單事件監聽器已綁定
   ✅ Event listeners attached
   ```

4. **選擇一個檔案**
5. **觀察 Console** 應該會看到：
   ```
   🔔 檔案下拉選單變更事件觸發！選擇的檔案: xxx.csv
   📂 準備調用 loadSimulationFile...
   📂 === loadSimulationFile 被調用 ===
   📂 檔案名稱: xxx.csv
   📂 Session ID: Mantle
   📂 發送 API 請求到 /api/simulator/load_file...
   📂 API 回應狀態: 200 OK
   📂 API 回應結果: {status: 'success', ...}
   ✅ 檔案載入成功！
   📂 === loadSimulationFile 結束 ===
   ```

6. **同時會彈出提示框**：
   ```
   ✅ 已載入模擬檔案:
   xxx.csv
   (244 筆數據)
   ```

## 🔍 如何判斷問題

### 情況 1：沒有看到「📂 File select element: 找到」
→ 表示 HTML 元素還沒載入完成
→ 解決：等待更久一點，或重新整理頁面

### 情況 2：看到「找到」但選擇檔案後沒有任何訊息
→ 表示事件監聽器沒有觸發
→ 請複製整個 Console 的內容給我

### 情況 3：看到「🔔 檔案下拉選單變更事件觸發」但沒有後續訊息
→ 表示 loadSimulationFile 函數沒有被執行
→ 請複製整個 Console 的內容給我

### 情況 4：看到「📂 API 回應狀態: 500」或其他錯誤
→ 表示後端 API 有問題
→ 請同時查看後端終端的錯誤訊息

### 情況 5：看到所有訊息都正常，但沒有彈出提示框
→ 這很奇怪，請截圖給我看

## 📸 如何截圖或複製內容

### 截圖 Console
1. 確保 Console 視窗可見
2. 按 Windows 鍵 + Shift + S（截圖工具）
3. 選擇 Console 區域
4. 貼到聊天中給我看

### 複製 Console 文字
1. 在 Console 中點擊右鍵
2. 選擇「全選」或「Select All」
3. 再次點擊右鍵，選擇「複製」或「Copy」
4. 貼到記事本或聊天中給我看

## ⚡ 快速測試（不用看 Console）

如果您不想看 Console，可以直接：

1. 重新整理頁面（F5）
2. 等待 3 秒
3. 選擇檔案
4. 看是否彈出「✅ 已載入模擬檔案」的提示框

**如果有彈出** → 問題解決了！
**如果沒有彈出** → 需要查看 Console，請按上面的步驟操作

## 🎯 簡易版（給不熟悉電腦操作的用戶）

1. **按 F12 鍵**
2. **點擊 Console**（在新出現的視窗上方）
3. **按 F5 重新整理頁面**
4. **等待 3 秒**
5. **選擇檔案**
6. **看 Console 裡面有沒有出現很多綠色的 ✅ 符號**

有綠色 ✅ → 好的
有紅色 ❌ → 有問題，截圖給我
什麼都沒有 → 事件沒觸發，截圖給我
